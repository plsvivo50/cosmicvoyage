# CosmicVoyage 开发指南 v4.4（联动扩展蓝图）

**基于 v4.3 工程冻结版 | 联动方向补充草案**
**Minecraft Java 1.20.1 Forge 47.4.0**
**联动目标：Create + MineColonies | 单人开发，单一核心Mod**

**重要声明：本文档不修改 v4.3 的冻结范围。v4.3 第一至第三阶段的内容、时间表和 Kill Criteria 保持完全有效。本文档仅作为"条件触发扩展包"的技术蓝图，在"社区需求 + 核心稳定"满足时启动评审。**

## 【零、v4.4 核心愿景】

### 0.1 一句话定位

在 v4.3 "沉浸式星际航行体验核心"之上，构建**"有人情味的异步星际文明网络"**——殖民地活在数据层，玩家在星空中飞翔。

### 0.2 核心哲学

> **殖民是游牧的。飞船是帐篷。星球是牧场。**

一切殖民系统的存在目的只有一个：**让玩家更想飞。**

不是管理，是牵挂。不是定居，是连接。不是扎根，是再启航。

### 0.3 v4.4 新增设计铁律

**6. 异步优先原则**
殖民地和机械设施在玩家不在场时必须以数据态自动运转。禁止要求玩家"留在地面监督"。

**7. 面板即终端原则**
所有殖民地/机械交互必须通过飞船面板完成，交互时长不得超过 15 秒。禁止打开子菜单、标签页或管理界面。

**8. 实体按需原则**
殖民地和 Create 设施仅在玩家降落时实例化为实体态。玩家起飞后必须回归数据态。禁止后台常驻实体加载。

**9. 视觉即反馈原则**
所有自动化流程必须有视觉表现（NPC 操作动画、机械运转、弹射舱飞行），但玩家仅需观看，无需操作。

**10. 二选一决策原则**
面板上的任何决策必须是二元选择（是/否，A/B）。禁止资源调配滑块、数字输入、优先级排序等复杂操作。

## 【一、异步殖民地架构：三态模型】

### 1.1 三态定义

| 状态 | 内存占用 | 运算开销 | 玩家可见性 | 切换条件 |
|------|---------|---------|-----------|---------|
| **休眠态** | ~2KB（纯数据文件） | 0 | 星图上一个光点 | 殖民地首次被发现 |
| **数据态** | ~10KB（JSON + 算法） | ~50 FLOP/tick | 飞船面板可读 | 殖民地建立后默认状态 |
| **实体态** | ~5-50MB（区块+实体+方块） | 完整物理模拟 | 完全可交互、可进入 | 玩家降落在殖民地所在星球 |

### 1.2 状态流转

```
发现星球
   ↓
休眠态 ──[玩家建立前哨站]──→ 数据态
                                  ↓
                        [玩家降落在星球]
                                  ↓
实体态 ←────── 数据烘焙 ───────┘
   ↓
[玩家起飞]
   ↓
快照保存 → 回归数据态
```

### 1.3 数据态运转逻辑

每 20 game ticks（1秒），对每个数据态殖民地执行：

```
1. 人口更新
   population += growth_rate × happiness × housing_ratio × delta_time

2. 资源更新
   for each resource:
     net = production_rate - consumption_rate
     stock += net × delta_time
     if stock < critical_threshold:
       push_event("resource_shortage", severity = f(stock/capacity))

3. Create 机器模拟
   for each machine:
     fuel_level -= burn_rate × delta_time
     efficiency = base_efficiency × (fuel_level > 0.1 ? 1.0 : degradation_curve)
     stress_usage += random_variance(-5%, +5%)
     if stress_usage > capacity × 0.95:
       push_event("stress_warning")
     if random() < breakdown_probability × (1 - maintenance_level):
       push_event("machine_breakdown", severity = weighted_random())

4. 跨殖民地物流
   for each outgoing_route:
     if source.stock > reserve_threshold and transport_ready:
       transfer_amount = min(capacity, source.surplus)
       source.stock -= transfer_amount
       target.incoming_queue += {amount: transfer_amount, eta: distance/cruise_speed}

5. 事件生成
   if event_queue_empty and random() < ambient_event_probability:
     generate_ambient_event(type = ["npc_dialogue", "discovery", "milestone"])
```

**性能指标：100 个殖民地同时运转 < 0.1ms/tick（主线程）**

### 1.4 实体态烘焙流程

**降落时（数据 → 实体）：**
1. 读取殖民地数据快照
2. 如果地形未生成：调用星球地形生成器
3. 按建筑等级放置 MineColonies 蓝图方块
4. 按机器类型和状态放置 Create 设施，还原应力网络参数
5. 按职业分布生成 NPC 实体，赋予基础行为树
6. 恢复事件队列中的活跃事件为实体行为
7. 播放"殖民地苏醒"过渡动画（可选，2-3秒）

**起飞时（实体 → 数据）：**
1. 快照当前实体状态（NPC 位置、机器实时参数、方块变更）
2. 写回数据层，覆盖原有数据
3. 卸载殖民地区块和实体
4. 标记为数据态，恢复算法增长
5. 如果在起飞时玩家触发"加速修复"等面板决策，先执行决策再保存

## 【二、MineColonies 联动重设计：星际邮路】

### 2.1 NPC 角色重定义：船员，不是定居者

| 传统 MineColonies | CosmicVoyage 邮路版 |
|------------------|-------------------|
| 农夫在地里种田 | 导航员在驾驶舱帮你校准航线 |
| 铁匠在打铁 | 工程师在引擎室维护 Create 机械 |
| 居民在房子里睡觉 | 科学家在观测舱记录星球数据 |
| 卫兵巡逻城墙 | 通讯官在电台接收殖民地求助 |

**核心差异：NPC 是飞船的居民，不是星球的居民。**

殖民地设施由 NPC "通勤"维护——他们住在飞船的船员舱，降落后走出飞船建立临时营地，起飞前收营回船。

### 2.2 殖民地自运转逻辑

NPC 在殖民地中的行为全自动，玩家不干预：

```
NPC 日常循环（实体态下可见，数据态下抽象为算法）：
├── 晨：从飞船走出 / 从宿舍醒来
├── 上午：操作 Create 机器（采矿/加工/维护）
├── 中午：在公共区交流（生成 ambient 对话事件）
├── 下午：建造/升级设施（如果蓝图已批准）
├── 傍晚：资源入库，记录日志
└── 夜：返回飞船 / 宿舍
```

**数据态下的抽象：** NPC 的所有行为压缩为三个数值：
- `work_output_multiplier`（工作效率系数）
- `happiness`（幸福感，影响产出和事件生成）
- `skill_level`（技能等级，随时间缓慢增长）

### 2.3 求助事件系统

殖民地仅在玩家**飞行途中**通过无线电发送求助。求助分为三级：

| 级别 | 频率 | 玩家响应 | 忽略后果 |
|------|------|---------|---------|
| **信息** | 每 5-10 分钟 | 无需响应，纯叙事 | 无 |
| **请求** | 每 15-30 分钟 | 面板二选一决策 | NPC 自行解决，效率降低 10% |
| **紧急** | 每 1-2 小时 | 必须面板决策或亲自前往 | 设施降级，殖民地规模缩减 |

**面板交互示例（请求级）：**

```
[通讯终端 - 火星观测站]
━━━━━━━━━━━━━━━━━━━━━━━
工程师 Elena：
"尘暴损坏了太阳能阵列的转向轴承。
我们有余料，但需要 2 根黄铜轴来完成修复。

[送黄铜轴]  [让 NPC 自己想办法]  [亲自去看看]"
```

**交互限制：**
- 选项数量 ≤ 3
- 每个选项是单一动作，无子菜单
- 选择后面板自动关闭
- 总交互时间 ≤ 15 秒

## 【三、Create 联动重设计：飞船生命体】

### 3.1 飞船引擎：一次性设计决策

玩家在造船时选择动力核心，此决策永久影响飞行手感：

| 动力核心 | 应力特性 | 飞行手感 | 视觉风格 |
|---------|---------|---------|---------|
| **Water Wheel** | 中等应力，稳定输出 | 丝滑平稳，惯性柔和 | 木质水轮，黄铜管道 |
| **Windmill Bearing** | 高应力，启动延迟 | 飘逸灵动，巡航轻快 | 白色风帆，旋转机构 |
| **Furnace Engine** | 极高应力，需燃料 | 暴力推背，加速迅猛 | 工业锅炉，蒸汽喷射 |
| **Creative Motor** | 无限应力（创造模式）| 无限制 | 核心发光 |

**决策后，飞船自动建造引擎室。** 玩家可以走进去观察机械运转，但不需要手动搭建基础框架。

### 3.2 殖民地设施：玩家送零件，NPC 建造

殖民地请求建造新设施时，流程如下：

```
1. NPC 工程师提出蓝图请求（面板/无线电）
   "我们需要一座风力发电机。需要：6 风帆 + 2 传动轴。"

2. 玩家通过飞行获取/合成资源
   （星球表面探索、小行星采集、跨殖民地贸易）

3. 玩家降落后，把零件放入 Landing Cache Pad

4. NPC 自动取件，开始装配
   - 实体态下：可见 Create 机械臂动画
   - 数据态下：装配在下次烘焙时完成

5. 装配完成后，殖民地 stress_capacity += new_machine.output
   玩家下次路过时看到新设施运转
```

**Create 设施清单（殖民地）：**

| 设施 | 功能 | 视觉表现 |
|------|------|---------|
| **Windmill Generator** | 殖民地电力 | 风帆旋转，应力管道发光 |
| **Drill Rig** | 自动采矿 | 钻头旋转，传送带运矿 |
| **Solar Tracker** | 能源补充 | Bearing 缓慢追踪太阳 |
| **Atmospheric Processor** | 燃料精炼 | 管道网络脉动，蒸汽排放 |
| **Launch Catapult** | 跨星球物流 | 弹射舱发射动画 |
| **Docking Arm** | 飞船对接 | 机械臂伸缩，齿轮咬合 |

### 3.3 跨星球物流：Create 弹射舱网络

```
殖民地 A ──[Create 弹射舱]──→ 太空
                                    ↓
                              弹道飞行（星图可见光流）
                                    ↓
殖民地 B ──[Create 接收臂]──→ 自动分拣入库
```

**玩家可见的反馈：**
- 星图上殖民地之间有**光流线**（Relay Network 可视化）
- 巡航中偶尔看到弹射舱串掠过窗外
- 殖民地日志："收到来自月球前哨站的补给，小麦 x50"
- 面板物流视图：实时流量动画

**技术实现：** 纯数据态运算，零实体加载。弹射舱在太空中是 billboard 粒子效果，不是真实实体。

### 3.4 Create 求助事件

| 事件类型 | 面板提示 | 玩家选项 | 自动解决 |
|---------|---------|---------|---------|
| **应力失衡** | "引擎室应力分配异常" | 优先推进 / 优先护盾 | 2 分钟后自动平衡 |
| **燃料不足** | "Furnace Engine 燃料低于 20%" | 送燃料 / 切换备用模式 | 切换 Water Wheel 降级运行 |
| **轴承磨损** | "传送带轴承需要更换" | 送零件 / 降低运转速度 | 降低 throughput 20% |
| **蓝图请求** | "工程师设计了一座新设施" | 批准（消耗资源）/ 否决 | 无 |

### 4.2 硬性设计约束

- **选项数量 ≤ 3 个按钮**
- **每次交互 ≤ 15 秒**
- **无子菜单、无标签页、无滚动条**
- **无数字输入、无滑块、无拖拽**
- **决策类型 = 二元或三元选择**
- **关闭面板后，玩家看到的画面是星空/飞船驾驶舱**

### 4.3 语音交互（可选增强）

殖民地事件可以通过**语音 + 字幕**方式传递，无需打开面板：

> *"舰长，火星观测站报告。尘暴损坏了太阳能阵列，请求指示。"*

玩家语音回复或快捷键选择：
- F1："送零件。"
- F2："自己解决。"
- F3："等我过去。"

## 【五、巡航社交：飞行间隙的情感填充】

### 5.1 设计目标

Outer Wilds 用叙事填充飞行间隙。CosmicVoyage v4.4 用**人际关系**填充。

核心原则：**巡航时间 = 社交时间。** 但社交是轻量的、可跳过的、增强而非打断的。

### 5.2 巡航事件类型

| 事件 | 触发 | 形式 | 时长 |
|------|------|------|------|
| **NPC 对话** | 随机，每 2-3 分钟 | 语音 + 字幕，自动播放 | 10-20 秒 |
| **殖民地呼叫** | 异步事件 | 面板提示 + 语音 | 15 秒决策 |
| **科学发现** | 接近未探索区域 | NPC 科学家评论 | 5-10 秒 |
| **船员互动** | 玩家走进船员舱 | 可选触发，走近对话 | 自由 |
| **星图更新** | 殖民地数据变化 | 视觉反馈（光流闪烁）| 即时 |

### 5.3 NPC 对话示例

```
[巡航中，速度 500 格/秒]

导航员 Aria：
"按照当前航线，我们将在 90 秒后进入月球巨物窗口。
第一次看到的总是会呼吸停止几秒——别紧张，这很正常。"

[30 秒后]

工程师 Kael：
"引擎室运转正常。Water Wheel 的转速比标准高了 3%，
我微调了一下齿轮比，巡航应该会更稳。"

[接近月球]

通讯官 Lira：
"月球前哨站在呼叫。他们说上次你送的零件已经装好了，
工程师想亲自谢谢你。"
```

**关键：这些对话不是任务。不是必须听的。不是会错过的。** 它们只是让你感觉——**飞船里有活着的人。宇宙里有牵挂你的人。**

## 【六、新增 Kill Criteria（v4.4 评审用）】

**评审前提：v4.3 第三阶段完成，且社区需求信号明确。**

| 编号 | 终止条件 | 触发后行动 |
|------|---------|-----------|
| **KC-8** | 异步殖民地系统导致飞行心流断裂（测试者报告"频繁被打断"） | 减少事件频率 50%，取消紧急级事件 |
| **KC-9** | Create 联动使飞船建造复杂度超过 30 分钟 | 动力核心选择简化为二选一，隐藏高级参数 |
| **KC-10** | MineColonies NPC 导致殖民地实体态加载时间 > 10 秒 | 减少 NPC 数量上限，简化行为树 |
| **KC-11** | 三态切换导致殖民地状态丢失或不同步 | 放弃实体态，全部采用数据态 + 简化视觉 |
| **KC-12** | 面板交互平均时长 > 30 秒 | 削减面板功能，只保留"查看"和"一键响应" |
| **KC-13** | 殖民系统使内存占用增幅 > 1GB（100 殖民地场景） | 减少数据态更新频率，降级殖民地上限至 20 |

**KC 唯一激活窗口：v4.4 开发前 4 周。**

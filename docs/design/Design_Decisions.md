### 2.1 遗迹文本系统 — "星文" (Astral Script)

**灵感**：Outer Wilds 挪麦人文字系统

**设计**：
```
核心概念：玩家发现遗迹中的神秘图案，逐步学习解读

实现方式：
  1. 视觉层：遗迹中显示独特的几何图案（星文）
  2. 认知层：玩家 initially 看不懂，但随着收集更多遗迹碎片
     → 逐渐理解图案含义（游戏自动"翻译"为玩家语言）
  3. 叙事层：图案承载前人的记录、警告、发现

图案设计语言：
  - 用简单的几何形状组合（圆、三角、线、点）
  - 类似航海时代的星图 + 象形文字
  - 每个图案对应一个概念（行星、危险、氧气、飞行等）

多语言适配：
  - 图案是视觉语言，不依赖文字
  - 翻译文件只决定"解读后的文本"语言
  - 新增语言 = 新增翻译文件，无需修改图案资源

Outer Wilds 对比：
  - OW：挪麦文字是从右到左的树状结构，需要理解语法
  - CV：更简单直观，接近"象形图标"，降低学习成本
  - 但保留"逐渐理解"的成就感
```

**示例**：
```
玩家发现的数据板：
  [图案：圆+放射线+波浪线] + [图案：三角+向下箭头] + [图案：骷髅]
  
初次查看："未知星文..."

收集3个相关遗迹后自动解读：
  "太阳风暴... 警告... 死亡"
  
收集10个遗迹后完整解读：
  "第7次太阳风暴来临时，我们的防护罩失效了。
   所有人撤退到地下掩体。
   ——探险者日志，火星前哨站，第42日"
```

### 2.2 前哨站 — "轻量生存"设计哲学

**核心矛盾**：
```
真实 ←→ 简化
密封检测(真实)   vs   只要方块完整(简化)
电力网络(真实)   vs   每个设备独立供电(简化)
资源管道(真实)   vs   无线传输(简化)
```

**决策**：
```
设计原则：飞行是核心，建造是辅助

前哨站复杂度等级：
  ★☆☆☆☆ 极简模式（推荐）：
    - 放置基础框架 → 自动生成墙壁和屋顶
    - 放置功能方块 → 自动连接（无线逻辑）
    - 氧气：只要 indoor 就自动有（区域检测）
    - 电力：太阳能板 → 范围内所有设备自动供电
    - 建造耗时：5分钟搭建一个功能站
    
  ★★☆☆☆ 标准模式（可选配置）：
    - 需要手动建造墙壁（但不需要气密检测）
    - 设备需要电力线连接
    - 氧气需要通风管道
    - 建造耗时：15分钟
    
  ★★★☆☆ 硬核模式（联动 Create 等）：
    - 完全开放的建造系统
    - Create 的机械动力支持
    - MineColonies 的 NPC 管理
    - 建造耗时：数小时（但这是 Create 玩家的乐趣）

默认：极简模式
目标：玩家花5分钟建好前哨站，然后回到飞船继续飞行探索
```

**具体功能方块**：
```
气闸门 (Airlock Door)
  - 功能：进入/离开需要关闭，否则氧气泄漏
  - 简化：自动感应开关，不需要手动操作
  - 危险：两个门不能同时开（否则氧气跑光）

氧气生成器 (Oxygen Generator)
  - 功能：为前哨站提供氧气
  - 简化：放置后自动工作，不需要输入资源
  - 限制：只能覆盖有限范围（需要多个覆盖大空间）

太阳能板 (Solar Panel)
  - 功能：白天发电
  - 简化：范围内所有设备自动供电
  - 限制：夜晚不工作，需要电池

飞船对接端口 (Docking Port)
  - 功能：停靠飞船
  - 简化：靠近后自动对接
  - 效果：飞船自动补充氧气/燃料

储物舱 (Storage Module)
  - 功能：扩大存储空间
  - 简化：大号箱子，不需要物流系统

观测窗 (Observation Window)
  - 功能：装饰+情绪价值
  - 效果：玩家靠近时自动隐藏内部渲染，显示外部太空
  - 情感：看着巨大行星缓缓升起的震撼
```

### 2.3 废弃站修复 — "本体简化，联动详细"

**核心设计策略**：
```
Cosmic Voyage 本体：简化但完整的前哨站系统
Cosmic Voyage + 联动模组：深度的自动化/管理系统
```

**废弃站修复（本体）**：
```
流程：
  1. 探索发现废弃站（世界自动生成）
  2. 交互查看损坏部件
  3. 提供资源（铁锭 x10 + 铜锭 x5）
  4. 即时修复，功能启用

修复后效果：
  - 氧气生成器恢复工作
  - 通讯塔可以接收信号（指引到附近遗迹）
  - docking port 可用

设计意图：
  - 降低建造门槛（比从零建造省50%资源）
  - 提供叙事线索（每个废弃站都有日志）
  - 作为探索目标（"找到下一个废弃站"）
```

**联动接口设计**：
```
Create 联动：
  - 废弃站的氧气生成器 = Create 的机械动力机器
  - 可以用 Create 的管道系统输送氧气
  - 可以用 Create 的机械臂自动维护设备
  - 太阳能板 = Create 的发电机（需机械动力驱动）

MineColonies 联动：
  - 废弃站可以发展为 NPC 殖民地
  - NPC 宇航员可以帮助维护前哨站
  - NPC 研究员可以解读星文（加速翻译）
  - NPC 矿工可以自动采集星球资源

通用联动接口：
  - Forge 事件系统：废弃站修复时触发事件
  - API 接口：其他模组可以注册自定义修复配方
  - 标签系统：废弃站方块可以用其他模组的方块替换
```

**设计哲学体现**：
```
"本体提供完整但轻量的体验
 联动提供深度但可选的体验
 玩家选择自己的复杂度"
```

| 5 | 小行星带 | 暂时搁置（需实体物理系统） | 真实但技术限制 |

| 6b | 氧气惩罚 | 原版溺水式缓慢扣血 | 熟悉机制 |
| 6c | 温度/气压 | **纳入考量，暂不实现**（TODO） | 真实但复杂度高 |
| 7 | 飞船设计 | **可建造的移动基地实体** | 核心游戏循环 |
| 7a | 升级方式 | **添加式升级** — 新能力而非数值提升 | 飞行连贯性 |
| 7b | 核心系统 | 驾驶舱+引擎+氧气+气密外壳 = 必须 | 安全检测 |

| 8 | 危险系统 | **物理环境危险为主**，无传统战斗 | 真实 |
| 8a | 引力危险 | 太阳引源、巨行星洛希极限、阻尼感对抗 | 真实物理 |
| 8b | 飞船损坏 | 与环境互动结果，与系统联动 | 生存压力 |
| 8c | 生物存在 | 部分天体有神秘生物（木卫二/土卫六等） | 探索乐趣 |
| 8d | 生物原则 | 不是所有星球都有生物（火星无） | 真实 |
| 8e | 无武器系统 | 不需要激光/导弹等传统战斗 | 非战斗导向 |
| 9 | 核心循环 | **知识驱动（探索发现）为主** | Outer Wilds 式 |
| 9a | 本体定位 | 飞行模拟器 + 轻度建造/生存 + 重探索 | 视听体验优先 |
| 9b | 联动定位 | 三者比重相等 = 文明模拟器 | Create/MC 联动 |
| 9c | 建造深度 | 本体极简，联动后深度自动化 | 分层设计 |
| 9d | 生存深度 | 本体轻度，联动后完整系统 | 分层设计 |
| 10 | 多人游戏 | **与单人同等重要**，但实现复杂 | 社交+合作 |
| 10a | 设计原则 | 殖民帝国需要伙伴，不孤独 | 情感驱动 |
| 10b | 实现策略 | 纳入所有系统设计考虑，分阶段实现 | 架构预留 |
| 10c | 优先级 | Phase 2-3 架构预留，Phase 3+ 完整实现 | 长期规划 |
